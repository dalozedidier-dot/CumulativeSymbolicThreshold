# Taxonomie d'application ORI-C
## Domaines et instanciation des variables par échelle
Version 1.0 — 25 février 2026

---

## Condition minimale d'applicabilité

ORI-C devient pertinent dès que le système présente simultanément :

1. Un système ouvert hors-équilibre (flux + dissipation)
2. Des boucles de régulation (feedback)
3. Une intégration (couplages internes / coordination)
4. Une trace transmissible (gènes, épigénétique, mémoire, normes, code, protocoles…)
5. Des séries temporelles (même proxy) permettant de rendre falsifiable le basculement

Ces conditions sont cohérentes avec le protocole de falsifiabilité du dépôt (installation + démos + *real data workflow*).

## 1. Échelle moléculaire / pré-cellulaire (frontière du vivant)

### 1A. Réseaux réactionnels auto-catalytiques / protocellules

| Variable | Instanciation |
|----------|---------------|
| O(t) | Structure du réseau (connectivité, modularité), stabilité des motifs réactionnels |
| R(t) | Tolérance aux perturbations (température, pH, dilution), redondance des voies |
| I(t) | Couplage entre sous-réseaux (métabolisme ↔ membrane ↔ énergie) |
| Σ(t) | Stress environnemental vs capacité de maintien (disponibilité énergétique) |
| S(t) | « Mémoire chimique » / templates / polymères conservés |
| C(t) | Gain cumulatif : meilleure persistance/reproduction de motifs |

### 1B. Virus / viroïdes / prions (cas limites)

ORI-C requiert de définir le « système vivant » comme le couple **agent + hôte** (sans hôte, le cycle interne autonome est absent).

| Variable | Instanciation |
|----------|---------------|
| O, R, I | Organisation de la réplication + réponses de l'hôte + intégration écologie cellulaire |
| S(t) | Transmission : variants, quasispecies, réservoirs |
| C(t) | Changement de régime (endémisation, échappement immunitaire) |

## 5. Échelle écosystèmes (communautés → biomes → biosphère)

### 5A. Communautés locales (récifs, forêts, lacs)

| Variable | Instanciation |
|----------|---------------|
| O(t) | Structure trophique, réseaux d'interaction |
| R(t) | Diversité fonctionnelle, redondance, capacité de recolonisation |
| I(t) | Couplage sol–plantes–insectes, cycles biogéochimiques |
| Σ(t) | Pressions (température, acidification, pollution) − capacité de tampon |
| S(t) | Banques de graines, propagules, refuges, « mémoire écologique » |
| C(t) | Tipping point (eutrophisation, blanchissement corallien, savanisation) |

### 5B. Biosphère / systèmes Terre

Même logique à l'échelle des variables globales :
- Cycles carbone/azote, cryosphère, biodiversité
- S(t) → stock de trace (diversité, infrastructures biotiques)
- C(t) → régime planétaire

## 6. Échelle psycho-sociale (du groupe à la civilisation)

C'est à cette échelle que S(t) prend pleinement son sens « symbolique » : langage, normes, institutions.

### 6A. Groupes, organisations, institutions

| Variable | Instanciation |
|----------|---------------|
| O(t) | Structure de coordination (rôles, processus), efficacité opérationnelle |
| R(t) | Tolérance aux crises, redondances, capacité de décision sous stress |
| I(t) | Alignement inter-équipes, cohérence des récits, interopérabilité |
| Σ(t) | Demande (marché, conflit, charge) − capacité réelle (ressources, attention) |
| S(t) | Procédures, règles, culture, documentation, routines transmissibles |
| C(t) | Bascule cumulative (polarisation, bureaucratisation, innovation stabilisée) |

Cohérent avec l'usage « pilotes » sur séries économiques (FRED/CPI/énergie) documenté dans les workflows de données réelles.

## 7. Vivant artificiel / hybride (IA, robots, socio-technique)

Si l'on accepte « vivant » = système adaptatif ouvert avec boucles + transmission, ORI-C s'applique directement — en particulier aux systèmes humains–machines.

### 7A. Agents ML / systèmes IA en production

| Variable | Instanciation |
|----------|---------------|
| O(t) | Organisation interne (performances, calibration, latences, drift) |
| R(t) | Robustesse aux shifts de données, attaques, pannes |
| I(t) | Intégration multi-modules (RAG, outils, mémoire, orchestration), multi-agents |
| Σ(t) | Charge / complexité des demandes − capacité (budget compute, qualité, sécurité) |
| S(t) | Code, datasets versionnés, politiques, logs, prompts, bases de connaissance |
| C(t) | Saut cumulatif (nouveau régime d'usage, nouvelle norme d'interface, verrouillage) |

### 7B. Collectifs hybrides (équipes humains–robots, plateformes, villes « smart »)

Même logique : S(t) devient procédures, standards, infrastructures, conventions, « mémoire organisationnelle » outillée.

## Mini-checklist d'instanciation pour un nouveau système candidat

Pour chaque nouveau candidat, remplir le gabarit suivant avant toute collecte de données :

1. **Définir le système + frontières** — quoi est dedans / dehors ?
2. **Lister les contraintes** — énergie, matière, temps, attention, capital…
3. **Choisir 2–5 proxies** pour O, R, I (mesurables en série temporelle)
4. **Définir Cap(t)** (forme fixée ex ante) et une Demande D(t) → Σ(t) = max(0, D − Cap)
5. **Définir S(t)** (trace transmissible) et C(t) (variable d'ordre attribuable à S sur horizon T)
6. **Lancer les tests de bascule / cumul** et vérifier que le cumul vient de S(t), pas seulement de O–R–I, conformément au protocole T1–T8

> **Règle d'audit** : tout proxy, forme fonctionnelle, poids, fenêtre Δ et horizon T doivent être fixés ex ante dans `PREREG_TEMPLATE.md` avant toute observation.
