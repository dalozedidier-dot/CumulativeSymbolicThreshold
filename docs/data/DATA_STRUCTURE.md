# Data & Code Structure — Clarification

This document explains why two data directories (`03_Data/` and `data/`) and two code directories
(`04_Code/` and `scripts/`) coexist, and how they differ.

---

## Data directories

### `03_Data/` — Curated scientific datasets (version-controlled)

`03_Data/` follows the numbered-directory convention of the research project
(`01_Theory/`, `02_Protocol/`, …) and contains **datasets that are part of the
scientific protocol**:

| Subdirectory | Content |
|---|---|
| `03_Data/synthetic/` | Synthetic time-series generated by the protocol scripts |
| `03_Data/real/` | Real-world datasets organised by sector (bio, climate, finance, …) including bundles `v1` and `v2` |
| `03_Data/sector_*/` | Sector-specific pilot datasets |

These files are tracked by git (or recorded in `manifest.json`) and are referenced
directly by pipeline scripts in `04_Code/pipeline/`.

### `data/` — Runtime / downloaded / large-file assets

`data/` holds files that are **downloaded at runtime**, produced by CI jobs, or
too large / binary to commit alongside the source code:

| Subdirectory | Content |
|---|---|
| `data/qcc/` | QCC Brisbane StateProb reference artifacts (ZIP + CSV + PDF) |
| `data/bundles/` | Sector pilot bundle ZIPs (downloaded by `scripts/`) |
| `data/bundles_extracted/` | Extracted contents of sector bundles |
| `data/climate/`, `data/finance/`, `data/survey/` | Raw data pulled from FRED, Eurostat, Eurobarometer |
| `data/real_datasets_index.csv` | Index of all registered real datasets |

These files are typically git-ignored or managed through `manifest.json` /
`data_pack_manifest.json`.

**Rule of thumb:**
- Scientific, reproducible, version-controlled → `03_Data/`
- Downloaded, large binary, CI-generated → `data/`

---

## Code directories

### `04_Code/` — Scientific pipeline (numbered project convention)

`04_Code/` contains the **full scientific pipeline** as defined by the research
protocol:

| Subdirectory | Content |
|---|---|
| `04_Code/pipeline/` | All executable pipeline scripts (run_all_tests.py, run_ori_c_demo.py, sector runners, …) |
| `04_Code/tests/` | Pytest integration and unit tests for the pipeline |
| `04_Code/reporting/` | CI summary builders and manuscript asset generators |
| `04_Code/sector/` | Per-sector fetch, generate, and run scripts |
| `04_Code/configs/` | JSON run configurations |

These scripts are called directly by CI workflows and by the `run_all.py` /
`run_all_tests.py` convenience scripts.

### `scripts/` — Convenience entry-points and operational utilities

`scripts/` holds **standalone helper scripts** that are not part of the core
scientific pipeline:

| File | Purpose |
|---|---|
| `scripts/run_all.py` | Convenience wrapper: runs all demos and writes output to `05_Results/` |
| `scripts/run_all_tests.py` | Convenience wrapper: delegates to `04_Code/pipeline/run_all_tests.py` |
| `scripts/fetch_fred_monthly.py` | Download FRED monthly data into `data/` |
| `scripts/fetch_eurostat_monthly.py` | Download Eurostat monthly data |
| `scripts/resolve_real_datasets.py` | Resolve dataset references in the registry |
| `scripts/build_real_from_local.py` | Build real-data bundles from local uploads |
| `scripts/check_bundle_integrity.py` | Verify ZIP bundle SHA256 hashes |
| `scripts/validate_proxy_spec.py` | Validate `proxy_spec.json` files ex ante |
| `scripts/generate_fred_report_pdf.py` | Generate PDF summary from FRED results |
| `scripts/run_brisbane_stateprob_pipeline.py` | Run the full QCC Brisbane StateProb pipeline |

`run_all` and `run_all_tests` are also exposed as **pip console scripts**
(`oric-run-all`, `oric-run-tests`) after `pip install -e .`.
