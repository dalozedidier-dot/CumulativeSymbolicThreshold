Outil 01 · Format · Science expérimentale

118 éléments — grammaire fonctionnelle en développement

On confond trois choses différentes dans la même phrase : ce qu'un élément fait dans un système, ce qu'il permet de connaître, et où il intervient. Le problème n'est pas la chimie — c'est le format de description.

Ce projet prend les 118 éléments et sépare systématiquement trois niveaux. La même grammaire est ensuite appliquée aux symboles mathématiques : le ∫ et le Fe, traités avec le même gabarit. Deux lignes sont comparables. Tu peux trier, filtrer, croiser. Et surtout, tu vois où ça coince — là où la description était floue.

Niveau 1 · Fonction

Ce qu'il fait dans un système

Un verbe, une cible, une finalité. Pas de description narrative.

Si → conduit · semi-conducteur · commutation
Niveau 2 · Rôle épistémique

Ce qu'il permet de connaître

Sonde, modèle, contrainte, traceur. Quatre catégories.

Si → modèle · réponse aux dopants
Niveau 3 · Secteur

Où il intervient

Énergie, santé, défense, numérique. Pas des sous-disciplines.

Si → numérique · énergie · défense

Outil 02 · Théorie systémique · Outil logiciel open-source

NEP-LIMITE 2.0 — La Civilisation Amplificatrice

Théorie systémique de la limite dans les systèmes ultra-puissants (IA, finance, réseaux sociaux, énergie, défense). Outil logiciel associé : Amplification-Barometer v0.4.13.

Quand la puissance d'action (échelle, vitesse, automatisation) grandit beaucoup plus vite que la capacité de contrôle et d'arrêt, le système devient instable. Des effets secondaires s'accumulent, les marges de sécurité s'érodent, et on risque des emballements difficiles à stopper — flash crash, désinformation massive, blackout énergétique, escalade involontaire.

L'étude propose une méthode pour mesurer cette tension et imposer des limites intelligentes avant que ça déraille, sans bloquer l'innovation.

Outil 03 · Astronomie & séries temporelles · Open-source

AstroOracle — suite

Trois outils pour détecter ce qui ne rentre pas dans la grille : sources astronomiques inhabituelles, transitions de régime, anomalies cachées dans des catalogues.

03 · A

AstroOracle

Oracle à apprentissage actif pour le triage d'anomalies astronomiques. Récupère des cutouts SkyView, classe les candidats via score d'anomalie, acquisition et diversité. Interface CLI et web UI pour l'annotation, avec ré-entraînement automatique.

apprentissage actif · SkyView · CLI / web UI
annotation · ré-entraînement automatique

03 · B

ChaosTrace

Toolkit Python léger pour la détection de transitions de régime et d'anomalies cachées dans des séries temporelles multivariées haute fréquence. Reconstruction d'espace de phase, analyse RQA et métriques inspirées du chaos.

reconstruction d'espace de phase · RQA
Takens · Lyapunov proxy · haute fréquence

03 · C

AstroGraphAnomaly

Détection de sources astronomiques inhabituelles dans les données Gaia. Combine le scoring d'anomalies basé sur les features avec une analyse de graphe k-NN sur la sphère céleste. Explainability LIME et génération de prompts LLM.

Gaia · k-NN sur sphère céleste
LIME explainability · prompts LLM
